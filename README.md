# Universal AI CLI Environment

**A governance layer for local AI agents to enforce engineering discipline, optimizing for hardware constraints and project hygiene.**

> "Stop treating AI like a chatbot. Treat it like a junior engineer."

## üöÄ Overview

This repository contains the configuration and scripts for the **Universal AI Environment**. It transforms a standard CLI AI agent workflow into a structured, audit-ready engineering process. By injecting strict "Global System Instructions" and hygiene scripts, it forces AI models to adhere to a **Read -> Plan -> Build -> Verify -> Record** loop.

## ‚ú® Features

*   **Global Brain (`GEMINI.md` / `CLAUDE.md`):** Injects a "Constitution" into the AI's context window, enforcing strict execution invariants.
*   **Hardware Protocol:** Optimized for consumer hardware (e.g., i7, 32GB RAM) by preventing full-drive scans and enforcing explicit file loading.
*   **Auto-Hygiene (`ai-auto-clean`):** A background daemon that monitors project dormancy and archives/deletes AI context (`.ai` folders) that haven't been touched in 30 days.
*   **Quality Gates (`verify.ps1`):** A local pre-commit hook that ensures worklogs are updated before a task is considered complete.

## ‚öñÔ∏è Pros & Cons

### ‚úÖ Pros
*   **Epistemic Integrity:** Eliminates "hallucinated on my machine" issues by forcing the AI to read context before acting.
*   **Disk Hygiene:** Automatically cleans up the massive amount of temporary context files and logs generated by AI tools, preventing disk bloat.
*   **Standardization:** Every project follows the same `WORKLOG.md` and architecture structure.
*   **Resource Efficiency:** explicitly prevents the AI from scanning `node_modules` or entire drives, saving CPU cycles.

### ‚ùå Cons
*   **Strictness:** The enforced "Read -> Plan" workflow can feel slow for very simple, one-off questions.
*   **Platform Lock:** Currently designed specifically for **Windows** (PowerShell Core / Windows PowerShell).
*   **Intrusive:** The installer modifies your global PowerShell profile to add aliases (`ai-init`, `ai-clean`).
*   **Data Deletion:** The hygiene script **DELETEs** files (specifically `.ai` directories) after 30 days of inactivity. While it creates an archive first, automated deletion always carries risk.
